Problem-solving when you're faced with JavaScript

Let me tell you about a time I got stuck on a simple problem: At the time, the problem didn't look simple. It was likely a JavaScript lesson. Here is how I learned to figure it out:

  • Sigh
  • Rub my temples
  • Google the issue
  • Curse at my computer when Google yields nothing helpful
  • "Walk away" from the problem (aka watch YouTube, check Facebook, talk to a friend)
  • Come back to the problem
  • Try something, anything. Change a punctuation mark.
  • Realise that the issue was as little as an incorrect punctuation mark.
  • Feel silly.
  • Repeat this process again with another issue.

There's many a time in which if I just took a deep breath, cleared my head and re-attacked the issue, I could have had the answer much easier. And if an answer wasn't clear, at least the fact that I needed help would have been more clear.

Now, has there ever been a time I solved an issue in an elegant way? If we're talking in terms of elegant JavaScript, then the answer is not yet. Upon reflection, my learning needs right now is to write things in an inelegant manner because my maths and logic skills are weaker than my other skills. I figure I need to "go slow to go fast", so right now, the elegant JavaScript needs to take a backseat. Eventually, I'll be able to simplify and write clean, elegant code. But now, I need to write out the mess to understand it.

That said, I've done some elegant solutions in clearing up potential issues in a previous career. Those solutions I won't write here, but I can share over a beer sometime. Diplomacy can be a very elegant solution. Perhaps there are some parallels between diplomacy and elegant code. Someone needs to write that book. In those situations, I would:

  • Take a step back to understand the problem
  • Consider a few options to solve that issue
  • Perhaps "walk away" from the issue to see if I develop any strong reactions to any of the options. (Sometimes, your gut is really good as sussing things out.)
  • After "walking away", come back and select the simplest option. Simple measures have less steps to muck up.
  • Execute said option.
  • Assess the option to see if it solved the issue. If it did, hooray! If it didn't, loop back (a while loop, that is), until the solution is solved.

      We've had to do quite a bit with manipulating arrays this week. Here are what the following array functions do:

      • .map(): Map basically applies a function to every item in a list of items. Whatever the result is - that result makes up a new list [array]. The input list is an array; the answer is a new list [array].
      • .filter(): Filter is kind of self descriptive. Say you have a grocery list [array], and you only want the items that meet a certain condition (let's say you want a list of all the items you need from the dairy section). Applying filter with the condition you want (maybe 'dairy' will be in the item name) will return a new list [array] with those items that meet your condition.
      • .reduce(): Reduce takes everything in a list and sums it up in one value. Let's go back to our grocery list, and let's say we give a price to each item. Reduce will take the prices of all those items and return one total figure.
