Javascript language

Learning JavaScript is like learning another language: we're relearning how to put statements together, and if something is out of place or missing a punctuation sign, it makes no sense. Or the page doesn't do what you want it to do. Here's what I make of the following marks:

Parentheses (): Parentheses encase the parameters for how a function or loop should be executed.

Brackets []: Brackets encase arrays, which is an item that encases multiple variables in one ... thing. (Arrays are somewhat different to an object, that's another ... thing). You can see brackets in objects, when objects have multiple arrays as part of their data.

Curly brackets or the set sign {}: Curly brackets help define the start and end of a function. They encase blocks of code.

Single quotes '': Single quotes can be used in event handlers, when the event references an element in the DOM. You could use double quotes here too, but be consistent. Don't use both. (A little bird told me that JSON doesn't use single quotes. Stay tuned.)

Double quotes "": Double quotes encase strings. And when used with a number, it tells the function to deliver the answer almost like a number string.
